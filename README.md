# Лабораторная работа №3
## Экспертная система «Консультант по сотовой связи»

### Описание порядка выполнения

В ходе выполнения лабораторной работы были выполнены следующие шаги:

#### 1. Изучен теоретический материал по теме:
- принципы построения экспертных систем
- работа с внутренними базами данных в Prolog (динамические предикаты)
- механизмы логического вывода (прямая и обратная цепочка рассуждений)
- предикаты поиска и фильтрации (`findall`, `member`, `forall`)

#### 2. Разработана база знаний (файл kb.pl):
- создан динамический предикат `has_feature/2` для хранения функций телефонов
- создан динамический предикат `purpose_requires/2` для связи целей с функциями
- добавлены факты для 8 целей (общение, выход в интернет, мобильный офис, фотография, игры, просмотр видео, прослушивание музыки, работа с документами)
- добавлены факты для 8 моделей телефонов (от бюджетных до флагманских)

#### 3. Реализована логика экспертной системы (файл lab3.pl):
- разработан предикат `all_phones/1` для сбора всех моделей из базы знаний
- разработан предикат `collect_required_features/2` для преобразования целей в список необходимых функций
- разработан предикат `suitable_phone/2` для проверки, есть ли у телефона все требуемые функции
- разработан предикат `find_suitable_phones/2` для поиска всех подходящих моделей
- добавлены предикаты `init/0`, `set/2`, `get/2` для хранения ответов пользователя во внутренней памяти
- добавлена проверка корректности ввода (разрешены только ответы `1` и `2`)
- реализован адаптивный опрос: последующие вопросы зависят от предыдущих ответов, поэтому общее число вопросов меняется

#### 4. Реализован главный предикат `consultant/0`, выполняющий:
- вывод приветствия и инструкций для пользователя
- инициализацию внутренней памяти перед началом консультации
- адаптивный опрос пользователя о нужных целях
- вычисление требуемых функций на основе выбранных целей
- поиск подходящих моделей телефонов
- вывод рекомендаций (или сообщения об отсутствии подходящих моделей)

#### 5. Проведено тестирование программы на различных сценариях:
- выбор только одной цели (например, только «общение» → бюджетный телефон или Nokia)
- выбор нескольких совместимых целей (например, «фотография» + «просмотр видео» → Google Pixel или iPhone)
- выбор требовательных целей (например, «игры» + «музыка» + «документы» → только iPhone 16 Pro)
- выбор целей, для которых нет полностью подходящей модели (вывод сообщения об отсутствии рекомендаций)
